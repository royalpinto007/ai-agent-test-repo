# dolibarr-dev — accumulated lessons

Append-only, dated, concise, verified lessons learned while doing Dolibarr
development. Read this at the start of every task; append a bullet at the end of
a task when you learn something reusable (a Dolibarr convention, a non-obvious
API, a pitfall, a corrected assumption). Keep entries one or two lines. Dedupe
before adding. If a lesson contradicts the SKILL.md body, fix SKILL.md instead.

## Lessons

- 2026-06-02 — There are 350+ trigger action codes; never guess. Extract with `scan-code.sh triggers '' <FILTER>` and confirm the firing site with `scan-code.sh find <CODE>`.
- 2026-06-02 — Customer-invoice validation fires `BILL_VALIDATE` (fired in `compta/facture/class/facture.class.php`); creation `BILL_CREATE`, payment `BILL_PAYED`.
- 2026-06-02 — Schema changes via `amb_*` (Advance Module Builder) write files immediately but do NOT auto-apply to the live DB. `amb_add_field`/`_edit_field`/`_delete_field` rewrite the object class `$fields` and `sql/llx_*.sql` (CREATE TABLE) for fresh installs; the live table only changes on module re-activation/upgrade. Verify on disk and with `dol-db.sh describe <table>`.
- 2026-06-02 — Common Dolibarr-native APIs to reuse: `GETPOST()` for input, `price()`/`price2num()` for money, `dol_now()` for time, `$object->fetch()`/`CommonObject`, `$db->query()`/`$db->fetch_object()`, `dol_syslog()` for logs, `newToken()`/CSRF. Find any of them with `scan-code.sh find-api <name>`.
- 2026-06-02 — find-api locates core helpers; reuse price2num() for money parsing
- 2026-06-10 — llx_c_shipment_mode rows drive the standard Shipping Method dropdown (only active=1 shows); it has a module column — set module='<mymodule>' to mark owned rows and toggle active per-service to hide/show them. Verified on Dolibarr 20.
- 2026-06-10 — Form::selectShippingMethod (core) lists ALL active llx_c_shipment_mode rows with no entity filter, sorted by libelle; displays trans('SendingMethod'.strtoupper(code)) if the key exists, else falls back to libelle — so custom rows render fine without lang keys. Verified on Dolibarr 20.
- 2026-06-15 — Trigger file naming: interface_NN_modXxx_Name.class.php; loader (core/class/interfaces.class.php) derives class as "Interface".ucfirst(Name) and qualifies the module via preg_replace('/^mod/i','',reg2) then isModEnabled(strtolower(...)) — so the modXxx segment must contain NO underscores. Verified Dolibarr 20.
- 2026-06-15 — The amb_* Module Builder MCP API can return HTTP 501 (endpoint unavailable). Fallback: hand-scaffold the module by mirroring an existing custom module descriptor (core/modules/modX.class.php with module_parts['triggers']=1, config_page_url, langfiles) — output is identical standard Dolibarr files.
- 2026-06-22 — FedEx multi-piece (multiple boxes): send one entry per box in requestedShipment.requestedPackageLineItems (each {groupPackageCount:1, weight, optional dimensions, dims only when all 3 > 0}) and set requestedShipment.totalPackageCount = box count, in BOTH the Rate and Ship payloads. The ship API returns one pieceResponse (one label + trackingNumber) per box plus a masterTrackingNumber; iterate pieceResponses to collect ALL labels/tracking — don't read [0]. Dolibarr expedition has ONE tracking_number column — store the master (or comma-join piece numbers). Verified in deliverycarrier, Dolibarr 20.
- 2026-06-22 — In deliverycarrier the rate form's `deliverycarrier_rate_request` array is persisted to llx_deliverycarrier_ratestate.request_json and reused at label time (generateLabel sets $object->deliverycarrier_rate_request = selected request), so any field added to the rate request (e.g. packages[]) automatically flows into the ship payload too — design rate + ship to read the same shape, keep flat fields (weight_value/length/...) mirroring box 1 for backward-compat.
- 2026-06-22 — Extrafields (extra attrs on a standard object, no custom table): API verified in core/class/commonobject.class.php — read via `$object->array_options['options_<name>']` (call `fetch_optionals()` if not loaded), write with `insertExtraFields()` (create) / `updateExtraField('<name>')` (update); columns live in llx_<element>_extrafields, defs in llx_extrafields; `amb_add_extrafields` MCP tool seeds defs. Use for a few per-instance attributes shown on the card; use a custom table for repeating/relational data. Live-DB caveat applies (appears on reactivation).
- 2026-06-22 — CLI-test private methods of a CommonObject subclass without HTTP: `require htdocs/master.inc.php` (it auto-detects php_sapi CLI), then ReflectionMethod->setAccessible(true) on a real instance. GETPOST('x','array') works after bootstrap if you set $_POST/$_REQUEST. Pitfall: max(0,$float) returns int 0 when equal (first arg wins) — use max(0.0,$x) for consistent float typing.
- 2026-06-21 — Theme CSS cache-bust: custom theme style.css.php sends Cache-Control: no-store (always), and its URL carries &revision=<MAIN_IHM_PARAMS_REV>. After editing a theme CSS file, browsers (esp. normal, non-incognito) can still show stale CSS; force-refresh ALL clients by bumping MAIN_IHM_PARAMS_REV per-entity (dolibarr_set_const / UPDATE llx_const) — this is what ?dol_resetcache=1 does server-side (main.inc.php). MAIN_OPTIMIZE_SPEED bit 0x02 = translation/memory cache only; does NOT cache CSS. Verified Dolibarr 22/20.
- 2026-06-24 — amb_* / Module Builder scaffolds a numbering model (core/modules/<module>/mod_<module>_standard.php) but leaves the descriptor `module_parts['models'] = 0` by default. Result: getNextNumRef can't load it ("Failed to include file mod_<module>_standard.php") and validate() leaves ref as (PROVx) — auto-numbering silently broken. For ANY module with ref auto-numbering, set `module_parts['models'] => 1` in the descriptor AND reactivate the module (the models dir registers via the MAIN_MODULE_<MODULE>_MODELS const written at activation, not read live from the file). Verify: validate a draft record → ref must become the real mask (e.g. GP2606-0001), not (PROVx).
